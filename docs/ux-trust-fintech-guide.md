# Pacto UX 가이드 — 신뢰·금융·모바일 퍼스트

> **목적:** 블로거 B2C 앱(캠페인·에스크로·지갑)과 B2B 관리자 화면에 공통으로 적용할 UX 원칙·벤치마크·실행 체크리스트를 정리한다.  
> **전제:** Pacto는 “돈이 오가는 약속(에스크로)” 플랫폼이므로, 단순 마켓플레이스 UX에 **은행·핀테크 앱의 신뢰·확인·투명성** 패턴을 반드시 결합한다.

**관련 문서:** `structure.md`(토스 스타일 UI), `api.md`(API 계약)

---

## 1. Pacto 서비스 맥락

| 역할 | 앱 | 핵심 행동 | UX에서 지켜야 할 것 |
|------|-----|-----------|---------------------|
| 블로거 | `b2c-web` | 캠페인 탐색 → 미션 수락 → URL 제출 → 정산 수령 → 출금 | “언제·얼마를 받는지” 항상 명확 |
| 광고주 | `admin-web` | 예치금 충전 → 캠페인 생성 → 승인/거절 | “돈이 어디 묶여 있는지” 가시화 |
| 대행사 | `admin-web` | 캠페인·지원자 관리 | 실수 방지·되돌리기 어려운 액션은 2단계 확인 |

**브랜드 포지션:** “거래를 약속대로” → 카피·화면 구조 전반에서 **약속(조건) · 잠금(에스크로) · 이행(제출) · 해제(정산)** 단계가 보이게 설계한다.

## 2. 벤치마크 리서치 요약

### 2.1 슈퍼앱·핀테크 (토스, 카카오페이, 네이버페이)

| 패턴 | 대표 사례 | Pacto 적용 |
|------|-----------|------------|
| 브랜드 스플래시 → 가치 제안 → 행동 유도 | 토스 첫 실행, 카카오톡 온보딩 | ✅ 적용됨: 스플래시 → 웰컴 → 로그인 선택 |
| 숫자·금액은 크고 굵게, 보조 설명은 회색 | 토스 홈 잔액 | 지갑 Hero, 캠페인 보상 금액 통일 |
| 카드형 정보 구획 (회색 배경 + 흰 카드) | 토스 전 화면 | 탐색·미션·지갑·설정 동일 레이어 |
| 하단 탭 3~5개, 핵심 IA 고정 | 토스·카카오 | 탐색 / 진행중 / 지갑 / 프로필 유지 |
| 로딩·빈 화면도 “다음 행동” 제시 | 토스 Empty | 미션 0건 → “캠페인 찾기” CTA |
| 에러는 짧은 문장 + 재시도 | 공통 | BFF/API 실패 시 재시도 버튼 |

### 2.2 소셜·마켓플레이스 (인스타그램, 당근, 배민)

| 패턴 | Pacto 적용 |
|------|------------|
| 탐색 피드 + 필터·정렬 | `explore` 카테고리·정렬 유지 |
| 상세에서 하단 고정 CTA 1개 | 캠페인 상세 `FixedBottomCTA` |
| 진행 상태 탭 (진행중/완료) | `missions` 탭·배지 |
| 로그인/가입 분리, 가치 먼저 | 웰컴 화면 유지 |

### 2.3 은행·디지털뱅킹 (KB스타뱅킹, 신한 SOL, 카카오뱅크, 토스뱅크, 우리 WON)

금융 앱은 **“예쁨”보다 “안심·통제·추적 가능”**이 우선이다.

| 원칙 | 은행 앱에서의 구현 | Pacto 매핑 |
|------|-------------------|------------|
| **가용 vs 불가 금액 분리** | 출금 가능 잔액 / 거래중 / 예금 등 구분 | `balance` vs `locked`(에스크로 잠금) 분리 표시 |
| **거래 단위 추적** | 거래 ID, 일시, 상대, 상태 | 미션·내역에 `escrow_id`, 캠페인명, `LOCKED→RELEASED` |
| **상태 라벨 표준화** | 이체완료·처리중·실패 | `PENDING` `LOCKED` `RELEASED` `CANCELLED` 한글+색 통일 |
| **고위험 액션 2단계 확인** | 이체 전 요약 모달 + 비밀번호/생체 | 출금·미션 수락·대량 충전 전 확인 시트 |
| **금액 입력 실수 방지** | 천 단위 콤마, 프리셋, 최소/최대 안내 | 출금 1만 원 최소, 잔액 초과 비활성 |
| **처리 중 피드백** | “접수됨 → 처리중 → 완료” 타임라인 | 출금 PENDING 후 내역·푸시(향후) |
| **사기·주의 고지** | 송금 전 경고, 공식 채널 안내 | 첫 출금·첫 미션 수락 시 에스크로 설명 1회 |
| **민감 정보 마스킹** | 계좌 끝 4자리만 | 프로필·계좌 표시 시 마스킹 |
| **세션·재인증** | 이체 시 재로그인/생체 | 출금 시 비밀번호 재확인(백엔드 지원 시) |
| **고객센터·분쟁 경로** | 거래 상세에서 문의 | 미션/내역 상세에 “문의·분쟁” 링크(향후) |

**은행 앱이 Pacto에 특히 중요한 이유:** 블로거는 “광고비를 받는 일”이지만 심리적으로는 **“내 돈이 맞는지”**와 동일하게 인식한다. 에스크로·출금 UX는 **이체 앱 수준의 명확성**을 목표로 한다.

## 3. 전사 UX 원칙 (B2C + Admin)

### 3.1 TRUST 원칙 (신뢰 5요소)

| 글자 | 의미 | UI 표현 |
|------|------|---------|
| **T**ransparency | 무슨 일이 일어나는지 투명 | 상태·수수료·예상 정산일 표시 |
| **R**eversibility 인지 | 되돌릴 수 있는지/없는지 | 취소·환불 불가 시 사전 고지 |
| **U**niformity | 앱 전반 동일 패턴 | 금액·날짜·상태 색·카드 스타일 통일 |
| **S**afety | 실수 방지 | 확인 모달, 금액 검증, 비활성 CTA |
| **T**rackability | 나중에 찾을 수 있음 | 내역·미션 목록·필터·검색 |

### 3.2 모바일 퍼스트 공통 규칙 (`b2c-web`)

- **최대 너비 480px**, PC는 레터박스(`#e5e8eb`) — 이미 적용
- **배경 `#F2F4F6`**, 콘텐츠 **흰 카드 `rounded-2xl`**, 그림자 `0 4px 20px rgba(0,0,0,0.04)`
- **Primary `#3182F6`** — CTA·링크·진행 강조만, 남용 금지
- **금액:** `text-2xl~3xl font-bold tabular-nums` + `원` 단위
- **터치 타겟:** 최소 44px 높이 (버튼·탭)
- **하단 탭 + 상단 고정 헤더** — 스크롤 영역만 움직임

### 3.3 “돈”이 걸린 화면 추가 규칙

1. **항상 3줄 요약:** 무엇을 / 얼마를 / 언제(또는 어떤 상태로)
2. **CTA 문구는 결과 지향:** “출금 신청하기” ✅ / “확인” ❌ (모호)
3. **성공 후:** 다음 행동 1개 (내역 보기, 탐색으로)
4. **실패 후:** 이유 + 해결 (잔액 부족 → 충전/미션 안내)

## 4. 화면별 UX 가이드 & Pacto 적용 상태

### 4.1 온보딩·인증 (`/login`, `/signup`)

**권장 흐름 (은행·토스 합성):**

```
스플래시(브랜드) → 웰컴(가치 3줄) → [로그인하기] → 로그인 폼
                              └→ [회원가입] → 가입 폼
```

| 항목 | 권장 | 현재 b2c-web |
|------|------|----------------|
| 폼 즉시 노출 | ❌ | ✅ 웰컴 후 선택 |
| 가입/로그인 분리 | ✅ | ✅ |
| 스플래시 매 방문 | 선택(브랜드 강화 vs 속도) | ✅ 매 `/login` |
| 회원가입도 웰컴 톤 통일 | ✅ | ✅ `AuthScreenLayout` |
| 데모 안내 문구 | 설정/푸터로 이동 | 로그인 폼 하단 (정리 권장) |

**추가 권장 (P1):**

- 회원가입 완료 후 **“첫 캠페인 찾기”** CTA로 `/explore` (은행 “첫 송금 안내”와 유사)
- 비밀번호 찾기·이용약관·개인정보 링크 (금융 앱 필수 요소)

---

### 4.2 캠페인 탐색 (`/explore`)

**벤치마크:** 배민 홈, 토스 혜택 탭, 당근 피드

| 권장 | 적용 |
|------|------|
| 카드에 **보상 금액** 최상위 시각 위계 | `CampaignListCard` |
| 마감·잔여 슬롯 **진행 바** | ProgressBar |
| 필터·정렬 변경 시 **스켈레ton** | 로딩 스피너 → 스켈레ton 권장 |
| 빈 목록 **일러스트 + CTA** | 카테고리 empty copy |

**신뢰 카피 (카드·상세 공통):**

- “미션 완료 시 에스크로에서 정산됩니다”
- “광고주 승인 후 지갑에 반영”

---

### 4.3 캠페인 상세 (`/campaigns/[id]`)

**벤치마크:** 은행 상품 상세(조건 표), 토스 송금 전 확인

| 권장 | 적용 |
|------|------|
| 보상·마감·슬롯 **요약 박스** 상단 | 부분 적용 |
| 미션 가이드 **체크리스트** | `MissionGuideline` |
| 수락 전 **확인 시트** (보상·마감·에스크로 안내) | ⚠️ 권장 (P0) |
| 하단 **단일 CTA** “미션 수락하기” | `FixedBottomCTA` |
| 마감/마감임박 **배지 색** (주의=amber) | hot/closed 처리 |

**미션 수락 확인 시트 예시 문구:**

> 강남 맛집 캠페인에 참여할까요?  
> · 보상: 50,000원 (승인 후 지갑 적립)  
> · 마감: 2026.05.25  
> · 진행 중에는 에스크로에 금액이 잠겨 있어요.

### 4.4 내 미션 (`/missions`)

**벤치마크:** 택배 추적, 은행 이체 처리현황

| 상태 | 사용자 질문 | UI |
|------|-------------|-----|
| LOCKED | 지금 뭐 하면 되지? | “URL 제출하기” 강조 |
| 검수 중 | 언제 돈 들어오지? | “광고주 승인 대기 중” + 예상 안내 |
| RELEASED | 받았나? | “정산 완료” + 지갑 링크 |
| CANCELLED | 왜? | 사유(가능 시) + 문의 |

| 권장 | 적용 |
|------|------|
| 탭: 진행중 / 완료 | 확인 필요 |
| Empty → 탐색 CTA | structure.md 설계 |
| URL 제출 **유효성·성공 토스트** | API 연동 |
| 카드에 **상태 칩** 색상 통일 | ⚠️ 표준화 권장 (P0) |

**상태 칩 색 제안:**

| API 상태 | 라벨 | 색 |
|----------|------|-----|
| LOCKED | 진행 중 | `primary` |
| PENDING (출금 등) | 처리 중 | `amber` |
| RELEASED | 정산 완료 | `green` |
| CANCELLED | 취소됨 | `muted` |

### 4.5 지갑 (`/wallet`) — **은행 앱 수준 적용 최우선**

**벤치마크:** 토스 송금·잔액, 카카오뱅크 거래내역, KB 이체 확인

| 권장 | 현재 | 우선순위 |
|------|------|----------|
| **출금 가능** vs **에스크로 잠금** 분리 표시 | 잔액만 | **P0** |
| 내역 필터 (전체/입금/출금) | ✅ | — |
| 출금 **2단계**: 금액 입력 → 확인 시트 | 인라인 폼 | **P0** |
| 금액 **천 단위 콤마** 실시간 | 숫자 파싱만 | P1 |
| 프리셋 (+1만 / 전액) | 없음 | P1 |
| 출금 완료 **성공 화면** + 내역 강조 | 토스트성 문구 | P1 |
| 거래 행: 날짜·제목·금액(+/-) 색 | ✅ | — |
| 입금 = primary+, 출금 = foreground- | 확인 | P1 |
| API `source: mock` 배지 (개발용) | 일부 | dev only |

**지갑 Hero 레이아웃 (권장 와이어):**

```
┌─────────────────────────────┐
│ 출금 가능          120,000원 │  ← 크게
│ 에스크로 잠금       50,000원 │  ← 작게, ? 툴팁
│ [ 출금 신청하기 ]            │
└─────────────────────────────┘
```

**출금 확인 시트 (은행 이체 확인 패턴):**

```
출금을 신청할까요?
출금 금액    50,000원
입금 계좌    OO은행 ****1234
수수료       0원
잔액 after   70,000원
[ 취소 ]  [ 신청하기 ]
```

## 5. 컴포넌트·디자인 토큰 체크리스트

프로젝트 전역에서 재사용할 공통 UI (신규/정리 권장):

| 컴포넌트 | 용도 | 참고 |
|----------|------|------|
| `AuthScreenLayout` / `AuthCard` | 로그인·가입 | ✅ |
| `BrandSplash` / `WelcomeScreen` | 온보딩 | ✅ |
| `MoneyAmount` | 금액 표시 (콤마·원·크기 variant) | ⚠️ 신규 권장 |
| `StatusChip` | 미션·출금·에스크로 상태 | ⚠️ 신규 권장 |
| `ConfirmSheet` | 수락·출금·충전 확인 | ⚠️ 신규 권장 |
| `EmptyState` | 탐색·미션·내역 | structure.md |
| `TransactionRow` | 지갑 내역 행 | 통일 |
| `TrustNotice` | 에스크로/출금 1회성 안내 | ⚠️ 신규 권장 |

**토큰 (이미 `globals.css`):**

```css
--toss-blue: #3182f6;
--toss-bg: #f2f4f6;
--foreground: #191f28;
--muted-foreground: #8b95a1;
```

**금액/상태 색 (추가 권장 CSS 변수):**

```css
--money-positive: #3182f6;   /* 입금·보상 */
--money-negative: #191f28;   /* 출금 */
--status-warning: #f59e0b;   /* 처리중 */
--status-success: #16a34a;   /* 완료 */
```

## 6. 마이크로카피 가이드 (한국어)

| 상황 | ❌ 피하기 | ✅ 권장 |
|------|-----------|---------|
| 에스크로 | “결제됨” | “광고주 예치금이 잠겨 있어요” |
| 정산 대기 | “대기” | “승인 후 지갑에 들어와요” |
| 출금 접수 | “완료” | “출금 신청을 받았어요 (처리 중)” |
| 오류 | “Error 400” | “최소 출금 금액은 10,000원이에요” |
| 빈 미션 | “데이터 없음” | “진행 중인 미션이 없어요” |

**톤:** 반말체(해요체) 유지 — 토스·카카오뱅크와 유사, 블로거 타깃에 친근하되 가벼우면 안 됨 → **지갑·출금만 존댓말 혼용 가능** (“신청하시겠어요?”)
